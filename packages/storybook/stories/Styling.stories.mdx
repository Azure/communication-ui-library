# Styling

The ability to style components using custom CSS is an important feature of Azure Communication UI Toolkit.
All the UI Components have complete CSS support and provide complex styling capabilities.

## Overview

- UI Components expose a `prop` named `styles` that accepts an Object containing custom CSS for the component root as well as it's child components.

- Users can always provide the `root` property in `styles` object and apply custom CSS to the component root.

- Properties in `styles` object will vary from component to component.

### Example

Let's style the `ControlBar` component and `DefaultButton` components inside it using the `root` styles property.

```tsx
import { ControlBar } from '@azure/communication-ui';

// ControlBar accepts a `styles` prop with only the `root` key as a valid property.
const styles = {
  root: {
    background: 'white',
    border: '2px solid firebrick',
    '& button': {
      ':hover': {
        background: 'black',
        color: 'white'
      }
    }
  }
}

<FluentThemeProvider>
  <ControlBar styles={styles}>
    <DefaultButton {...videoButtonProps} />
    <DefaultButton {...audioButtonProps} />
    <DefaultButton {...optionsButtonProps} />
    <DefaultButton {...hangupButtonProps} />
  </ControlBar>
</FluentThemeProvider>
```
