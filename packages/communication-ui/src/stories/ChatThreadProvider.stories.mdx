# ChatThreadProvider

The ChatThreadProvider sets up a chat thread for a specific ACS chat thread. The ChatThreadProvider is necessary but not sufficient for setting up acs chat threads.
The ChatThreadProvider needs to be used as a descendant under a ChatProvider to have the necessary context for an ACS chat thread. We have both ChatProvider and ChatThreadProvider
concepts since we wanted to distinguish state around the overall chat experience and the chat thread instance.

```jsx
<ChatProvider token="<TOKEN>", displayName="<DISPLAYNAME>", threadId="<THREADID>", endpointUrl="<ENDPOINTURL>">
   <ChatThreadProvider>
      // chat thread 1
   </ChatThreadProvider>
</ChatProvider>
```

## Importing

```javascript
import { ChatThreadProvider } from '@azure/communication-ui';
```

## Hooks

These hooks are dependant on the ChatThreadProvider

- useFetchMessage
- useFetchMessages
- useFetchReadReceipt
- useIsMessageSeen
- iseRemoveThreadMember
- useSendMessage
- useSendReadReceipt
- useSendTypingNotification

These hooks are dependant on ChatProvider and ChatThreadProvider

- useFetchThread
- useFetchThreadMembers
- useSubscribeMessage
- useSendTypingNotification

## Caveats

- The ChatThreadProvider must be a descendant of a ChatProvider
- \*Currently we only handle one ChatThreadProvider at a time
